<task type="auto">
  <name>Task 2: 实装 interrupt 队列与 session registry</name>
  <files>apps/gateway/src/control/queue.ts, apps/gateway/src/control/worker.ts, apps/gateway/src/session/registry.ts, infra/docker-compose.redis.yml</files>
  <action>
    - 采用 `bullmq` + Redis Streams，定义队列 `duplex-control`，control frame priority=0, audio/render priority=1。
    - `queue.ts` 负责 push interrupt job（payload: sessionId, ackId, ts）。
    - `worker.ts` 监听队列并：
      1. 更新 `sessionRegistry`（Map + TTL）；
      2. 调用 `abortCurrentStreams(sessionId)`；
      3. 回复 ACK（ws send ack message + timestamp）。
    - `session/registry.ts` 管理连接引用 +心跳（setInterval 清理过期 session）。
    - 在 `infra/docker-compose.redis.yml` 声明 redis:7 镜像 + 持久卷供本地测试。
    - WHY: 没有可靠队列就无法保证 control 优先级，也无法满足 <50ms ack 指标。
  </action>
  <verify>
    <automated>docker compose -f infra/docker-compose.redis.yml up -d redis && pnpm --filter gateway test control --runInBand</automated>
  </verify>
  <done>
    - 控制帧写入队列立刻触发 worker。
    - `sessionRegistry` 可在 5s 内回收断开连接。
  </done>
</task>
